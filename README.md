# REST-сервис системы каршеринга (Car Sharing Service)

Данный проект представляет собой разработку базового REST-сервиса для управления системой каршеринга. Сервис реализован с использованием фреймворка Spring Boot и демонстрирует принципы построения многослойной архитектуры, обработку HTTP-запросов с параметрами и использование DTO.

## Архитектура приложения

Проект реализует классическую многослойную архитектуру, состоящую из трех основных уровней:

1. Controller Layer (Уровень контроллеров):
- Принимает и обрабатывает входящие HTTP-запросы
- Выполняет валидацию входных параметров
- Использует @RequestParam для извлечения параметров запроса
- Использует @PathVariable для извлечения параметров из URL
- Преобразует DTO в сущности и обратно
- Возвращает ответы клиенту в формате JSON

2. Service Layer (Сервисный уровень):
- Содержит бизнес-логику приложения
- Взаимодействует с репозиторием для получения/сохранения данных
- Выполняет маппинг между сущностями и DTO
- Реализует правила аренды автомобилей

3. Repository Layer (Репозиторий):
- Обеспечивает доступ к данным
- В данной реализации использует in-memory хранилище (коллекции Java)
- Предоставляет CRUD операции над сущностями

## Модель данных

Сущность Car (Автомобиль):
- id — уникальный идентификатор автомобиля
- brand — марка автомобиля
- model — модель автомобиля
- licensePlate — государственный номер
- status — статус автомобиля (AVAILABLE, RENTED, MAINTENANCE)
- rentalPricePerHour — стоимость аренды в час


Data Transfer Objects (DTO):

Для изоляции клиентской части от внутренней модели данных используются два DTO:

*CarRequest* — используется для создания и обновления автомобиля. Содержит только поля, которые клиент может отправлять: brand, model, licensePlate, rentalPricePerHour. Не содержит id и status, так как они генерируются на сервере.

*CarResponse* — используется для ответа клиенту. Содержит полную информацию об автомобиле: id, brand, model, licensePlate, status, rentalPricePerHour.
